# 🤝 Cómo contribuir a este proyecto

¡Gracias por tu interés en mejorar el Escáner de Obsidian! Este es un proyecto de código abierto y cualquier contribución (desde reportar un error hasta proponer nuevas funcionalidades) es súper bienvenida.

Si estás dando tus primeros pasos en Python o en el uso de Git, este es un excelente lugar para practicar.

## 🛠️ Flujo de trabajo para contribuir

Para mantener el código ordenado, usamos el flujo de trabajo estándar de *Pull Requests*:

1. **Haz un Fork** de este repositorio hacia tu cuenta de GitHub.
2. **Clona** tu fork localmente en tu computadora.
3. **Crea una nueva rama** para tu funcionalidad o corrección:
`git checkout -b feature/nombre-de-tu-mejora`
4. **Escribe tu código**. Asegúrate de probarlo en una bóveda de Obsidian de prueba (¡no uses tus notas reales para testear!).
5. **Haz un Commit** de tus cambios con un mensaje claro:
`git commit -m "Añade soporte para leer PDFs"`
6. **Sube (Push)** la rama a tu fork en GitHub:
`git push origin feature/nombre-de-tu-mejora`
7. **Abre un Pull Request (PR)** en este repositorio original explicando qué cambiaste y por qué.

## 📝 Estilo de Código

* El proyecto está escrito en Python 3. Intenta mantener el código modular (usando funciones claras).
* Si agregas una función nueva, por favor añade un pequeño bloque de comentarios (docstring) explicando qué hace, qué recibe y qué devuelve.
* Si introduces nuevas expresiones regulares (`re`), documenta brevemente qué patrón estás buscando.

## 💡 ¿No sabes por dónde empezar?

Revisa la pestaña de **Issues** del repositorio. Busca aquellos que tengan la etiqueta `good first issue` (buen primer problema). ¡Son tareas ideales para arrancar!
